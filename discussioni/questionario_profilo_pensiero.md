# Questionario: Scopri il tuo Profilo di Pensiero

## Istruzioni

Questo questionario ti aiuta a scoprire come pensi — non quanto sei bravo a pensare. Non ci sono risposte giuste o sbagliate: ogni opzione descrive un modo diverso e ugualmente valido di affrontare le situazioni.

Per ogni scenario, scegli **le due risposte** che ti rappresentano di più. Se una sola ti rappresenta fortemente, scegli solo quella. Se nessuna ti convince, salta la domanda.

Rispondi in base a ciò che **fai davvero**, non a ciò che vorresti fare.

## SEZIONE A — Come nasce il pensiero (Canale cognitivo)

### A1. Stai cercando di capire un concetto nuovo e complesso. Mentre ci ragioni, cosa succede nella tua testa?

- **(a)** Mi parlo dentro: costruisco frasi, argomento con me stesso, formulo obiezioni silenziose → `D1: 1.1 Verbale interno`
- **(b)** Vedo immagini, schemi, configurazioni spaziali — il concetto prende forma visiva prima che verbale → `D1: 1.2 Visuo-spaziale`
- **(c)** Sento una sensazione fisica: tensione, apertura, disagio corporeo che mi dice se sono sulla strada giusta → `D1: 1.3 Somatico`
- **(d)** Il concetto ha un "ritmo" o una "melodia" — sento quando qualcosa stona o è armonioso → `D1: 1.4 Sonoro/ritmico`
- **(e)** Non c'è nulla di specifico: né parole, né immagini, né sensazioni. È un ragionamento puro, astratto, che non ha forma → `D1: 1.5 Astratto-proposizionale`

### A2. Ti svegli di notte con la soluzione a un problema su cui rimuginavi. In che forma si presenta?

- **(a)** Come una frase o un'argomentazione — "ma certo, il punto è che..." → `D1: 1.1`
- **(b)** Come un'immagine mentale o uno schema — "vedo" la soluzione → `D1: 1.2`
- **(c)** Come una certezza corporea — "lo sento" che è giusto, prima ancora di saperlo spiegare → `D1: 1.3`
- **(d)** Come una sensazione di "tutto torna" — un equilibrio, quasi musicale → `D1: 1.4`
- **(e)** Come un sapere improvviso senza forma — so la risposta ma non so come la sto pensando → `D1: 1.5`

### A3. Devi spiegare a qualcuno il funzionamento di un sistema complesso (un'organizzazione, un software, un ecosistema). Nella tua mente, quel sistema com'è rappresentato?

- **(a)** Come un racconto: "prima succede X, poi Y reagisce, quindi Z cambia..." → `D1: 1.1`
- **(b)** Come una mappa o un diagramma con componenti collegati da frecce e relazioni → `D1: 1.2`
- **(c)** Come qualcosa che "sento" funzionare — percepisco le tensioni tra le parti → `D1: 1.3`
- **(d)** Come un ritmo o un flusso — alcune parti sono veloci, altre lente, alcune in sincrono → `D1: 1.4`
- **(e)** Come un insieme di regole e relazioni astratte, senza bisogno di visualizzarlo → `D1: 1.5`

### A4. Stai ascoltando qualcuno che presenta un'idea. A un certo punto senti che qualcosa non torna. Come lo percepisci?

- **(a)** Formulo mentalmente l'obiezione: "sì, ma questo contraddice quello che hai detto prima..." → `D1: 1.1`
- **(b)** "Vedo" un pezzo del puzzle che non combacia, un'immagine che non quadra → `D1: 1.2`
- **(c)** Sento un disagio fisico: qualcosa nello stomaco, una tensione, un impulso a interrompere → `D1: 1.3`
- **(d)** L'argomentazione "stona" — come una nota sbagliata in una melodia → `D1: 1.4`
- **(e)** Individuo la falla logica senza che prenda una forma specifica → `D1: 1.5`

### A5. Devi prendere una decisione importante. Ti prendi un momento per riflettere. Cosa fai concretamente?

- **(a)** Dialogo con me stesso, elencando pro e contro a parole nella mia testa → `D1: 1.1`
- **(b)** Immagino visivamente gli scenari alternativi, come scene di un film → `D1: 1.2`
- **(c)** Mi chiedo "come mi sento rispetto a questa opzione?" e ascolto le sensazioni → `D1: 1.3`
- **(d)** Cerco quale opzione "suona bene" — ha il ritmo giusto, il tono giusto → `D1: 1.4`
- **(e)** Ragiono sulle implicazioni logiche di ciascuna opzione senza coinvolgere sensi o emozioni → `D1: 1.5`

## SEZIONE B — Come fai avanzare il pensiero (Strategia/dinamica)

### B1. Hai un'idea che ti sembra buona. Qual è il passo successivo?

- **(a)** La espongo a qualcuno che so che la smonterà — voglio vedere dove si rompe → `D2: 2.1 Adversariale`
- **(b)** Mi faccio domande: "perché penso questo? Quali assunzioni sto facendo? Cosa non sto considerando?" → `D2: 2.2 Socratico`
- **(c)** La metto in pratica in forma grezza, poi la raffino: primo tentativo, poi miglioro → `D2: 2.3 Iterativo`
- **(d)** Provo a spiegarla a qualcuno (o a me stesso) come se dovessi insegnarla → `D2: 2.4 Insegnamento`
- **(e)** La confronto con casi analoghi: "in altri contesti simili, come ha funzionato?" → `D2: 2.8 Analogia`

### B2. Affronti un sistema o un processo che non capisci. Come ti approcci?

- **(a)** Lo smonto pezzo per pezzo: tolgo componenti finché non capisco cosa fa ognuno → `D2: 2.5 Decostruttivo`
- **(b)** Raccolgo informazioni senza fretta — leggo, osservo, accumulo, finché non scatta qualcosa → `D2: 2.6 Accumulo e soglia`
- **(c)** Elimino le spiegazioni sbagliate una per una finché non resta quella giusta → `D2: 2.7 Eliminazione`
- **(d)** Cerco di tradurlo in un linguaggio o framework che già conosco → `D2: 2.11 Traduzione tra domini`
- **(e)** Cerco pattern: "questo mi ricorda qualcosa che ho già visto" → `D2: 2.14 Pattern recognition`

### B3. Sei in una riunione dove si discute una decisione difficile. Qual è il tuo contributo tipico?

- **(a)** Costruisco l'argomento contrario a quello prevalente, anche se non ci credo → `D2: 2.1 Adversariale`
- **(b)** Genero il maggior numero di alternative possibili, senza giudicarle → `D2: 2.10 Generativo/brainstorming`
- **(c)** Provo a mettermi nei panni delle persone coinvolte: "come la vedrebbe il cliente? E il fornitore?" → `D2: 2.17 Perspective-taking`
- **(d)** Osservo e mi chiedo "stiamo ragionando nel modo giusto? Stiamo usando l'approccio corretto per questo tipo di problema?" → `D2: 2.16 Metacognitivo`
- **(e)** Trasformo la discussione in una storia: "immaginiamo che tra un anno sia andata così..." → `D2: 2.12 Narrazione`

### B4. Un progetto a cui lavori non sta funzionando come previsto. Come reagisci?

- **(a)** Immagino che il progetto sia già fallito completamente e ricostruisco le cause a ritroso → `D2: 2.13 Simulazione mentale (pre-mortem)`
- **(b)** Cerco di competere con il problema: lo tratto come un avversario da battere, una sfida → `D2: 2.15 Competizione/gioco`
- **(c)** Riparto dalla versione base e itero: tolgo il superfluo, riprovo, aggiusto → `D2: 2.3 Iterativo`
- **(d)** Mi chiedo cosa mi dice l'istinto — ho già visto abbastanza problemi simili da avere un "colpo d'occhio" → `D2: 2.14 Pattern recognition`
- **(e)** Provo a smontare il problema nelle sue parti e capire quale pezzo non funziona → `D2: 2.5 Decostruttivo`

### B5. Devi capire un argomento nuovo di cui sai poco. Come studi?

- **(a)** Leggo e raccolgo materiale senza cercare di capire subito — accumulo finché non emerge una struttura → `D2: 2.6 Accumulo`
- **(b)** Cerco di spiegare subito quello che ho capito (anche male) a qualcuno o a me stesso → `D2: 2.4 Insegnamento`
- **(c)** Cerco qualcosa di simile che già conosco: "è come..." → `D2: 2.8 Analogia`
- **(d)** Formulo domande progressive: cosa? perché? come? cosa succederebbe se...? → `D2: 2.2 Socratico`
- **(e)** Traduco tutto in un framework che conosco (es. un problema di biologia letto in termini economici) → `D2: 2.11 Traduzione tra domini`

### B6. Qualcuno propone una soluzione che ti sembra sbagliata ma non sai perché. Come procedi?

- **(a)** Argomento contro, anche senza avere ancora un'alternativa — la critica farà emergere il problema → `D2: 2.1 Adversariale`
- **(b)** Costruisco mentalmente lo scenario in cui la soluzione viene implementata e cerco dove fallisce → `D2: 2.13 Simulazione mentale`
- **(c)** Mi chiedo: "quale assunzione nascosta sta facendo questa persona?" → `D2: 2.9 Contraddizione interna`
- **(d)** Cerco di assumere il punto di vista di chi propone: "perché lo vede così?" → `D2: 2.17 Perspective-taking`
- **(e)** Escludo una per una le ragioni per cui potrebbe essere giusta, per isolare il problema → `D2: 2.7 Eliminazione`

### B7. Come descrivi il tuo rapporto con le regole e i vincoli in un progetto creativo?

- **(a)** I vincoli mi aiutano: limiti di tempo, formato, risorse — penso meglio dentro confini → `D2: 2.3 Iterativo (auto-vincolo)`
- **(b)** Prima genero liberamente, poi applico i vincoli come filtro → `D2: 2.10 Generativo`
- **(c)** Li tratto come regole del gioco: parte del divertimento è trovare la soluzione dentro i limiti → `D2: 2.15 Competizione/gioco`
- **(d)** Li metto in discussione: "ma questo vincolo è davvero necessario?" → `D2: 2.9 Contraddizione interna`
- **(e)** Li traduco in un linguaggio formale: equazioni, codice, regole esplicite → `D2: 2.11 Traduzione tra domini (formalizzazione)`

## SEZIONE C — Dove va il tuo pensiero (Direzione)

### C1. Stai cercando di risolvere un problema. Come procedi?

- **(a)** Genero il più possibile di opzioni diverse prima di scegliere — esploro ampiamente → `D3: 3.1 Divergente`
- **(b)** Restringo rapidamente le opzioni ed elimino quelle meno promettenti — vado dritto al punto → `D3: 3.2 Convergente`
- **(c)** Cerco un'angolazione completamente diversa da quella ovvia — cambio prospettiva radicale → `D3: 3.3 Laterale`
- **(d)** Alterno: prima espando le possibilità, poi seleziono, poi espando di nuovo → `D3: 3.1+3.2 Ciclo divergente-convergente`

### C2. Ti presentano un caso specifico e ti chiedono un'opinione. La tua prima reazione è:

- **(a)** Cerco la regola generale: "questo caso è un'istanza di quale principio più ampio?" → `D3: 3.4 Astrattivo`
- **(b)** Cerco altri casi concreti simili: "quando è successo qualcosa del genere?" → `D3: 3.5 Concretizzante`
- **(c)** Mi chiedo "come apparirà tra 5 anni? E se fosse successo 10 anni fa?" → `D3: 3.8 Scala temporale`
- **(d)** Cerco cosa togliere: "qual è l'essenza del caso, spogliato di tutto il superfluo?" → `D3: 3.6 Sottrattivo`

### C3. Devi migliorare qualcosa che già funziona (un processo, un prodotto, una presentazione). Come lo affronti?

- **(a)** Aggiungo: nuove funzionalità, nuovi passaggi, nuovi dettagli → `D3: 3.1 Divergente (additivo)`
- **(b)** Tolgo: elimino passaggi inutili, semplifico, riduco al minimo → `D3: 3.6 Sottrattivo`
- **(c)** Comprimo: cerco di ottenere lo stesso risultato in meno spazio/tempo/parole → `D3: 3.7 Compressivo`
- **(d)** Cambio frame: lo guardo da un'angolazione completamente diversa per vedere cosa non vedevo → `D3: 3.3 Laterale`

### C4. Un amico ti chiede consiglio su un problema personale. Qual è la tua prima mossa mentale?

- **(a)** Cerco il principio: "il problema di fondo qui è..." → `D3: 3.4 Astrattivo`
- **(b)** Chiedo dettagli concreti: "raccontami un episodio specifico" → `D3: 3.5 Concretizzante`
- **(c)** Proietto nel tempo: "dove porti questa situazione tra un anno se non cambi nulla?" → `D3: 3.8 Scala temporale`
- **(d)** Genero opzioni: "potresti fare A, oppure B, oppure C, oppure..." → `D3: 3.1 Divergente`

### C5. Come giudichi la qualità di una spiegazione?

- **(a)** È buona se copre tutti i casi e le eccezioni — completezza → `D3: 3.1 Divergente`
- **(b)** È buona se arriva rapidamente al punto — efficienza → `D3: 3.2 Convergente`
- **(c)** È buona se sta in una frase — compattezza → `D3: 3.7 Compressivo`
- **(d)** È buona se funziona a livelli diversi — dal caso specifico al principio generale → `D3: 3.4+3.5 Astrattivo-concretizzante`

## SEZIONE D — Come fai uscire il pensiero (Medium di esternalizzazione)

### D1. Stai lavorando su un problema complesso da solo. Cosa fai per sbloccarti?

- **(a)** Ne parlo ad alta voce, anche se non c'è nessuno → `D4: 4.1 Voce, dest. = sé`
- **(b)** Apro un documento e comincio a scrivere i miei ragionamenti → `D4: 4.2 Scrittura, dest. = sé`
- **(c)** Prendo carta e penna e disegno uno schema, una mappa, un diagramma → `D4: 4.3 Diagramma, dest. = sé`
- **(d)** Costruisco qualcosa: un prototipo, un modello, anche solo con oggetti sulla scrivania → `D4: 4.4 Gesto/prototipo, dest. = sé`
- **(e)** Scrivo pseudocodice, equazioni, o traduco il problema in un linguaggio formale → `D4: 4.5 Codice/formalizzazione, dest. = sé`

### D2. Devi spiegare un'idea complessa a qualcuno. Come la presenti?

- **(a)** Gliela racconto a voce, ragionando insieme → `D4: 4.1 Voce, dest. = altro`
- **(b)** Scrivo un testo: email, documento, messaggio strutturato → `D4: 4.2 Scrittura, dest. = altro`
- **(c)** Disegno un diagramma o uno schema alla lavagna → `D4: 4.3 Diagramma, dest. = altro`
- **(d)** Faccio una demo o un prototipo: "guarda, funziona così" → `D4: 4.4 Gesto/prototipo, dest. = altro`
- **(e)** Formalizzo: specifiche, formule, codice, modelli → `D4: 4.5 Codice/formalizzazione, dest. = altro`

### D3. Per chi esternalizzi di più i tuoi pensieri?

- **(a)** Per me stesso: tengo note, diari, appunti che riletto → `D4 dest.: sé / sé futuro`
- **(b)** Per le persone con cui lavoro: comunico molto, spiego, discuto → `D4 dest.: altro umano`
- **(c)** Per me stesso in futuro: blog, documentazione, registrazioni che userò dopo → `D4 dest.: sé futuro`
- **(d)** Per sistemi: prompt per AI, codice, specifiche formali → `D4 dest.: sistema/LLM`

### D4. Ripensa a un momento in cui hai avuto un'intuizione importante. Come l'hai "fissata" per non perderla?

- **(a)** L'ho detta ad alta voce a qualcuno subito → `D4: 4.1 Voce`
- **(b)** Ho preso appunti scritti il prima possibile → `D4: 4.2 Scrittura`
- **(c)** Ho disegnato uno schizzo o schema rapido → `D4: 4.3 Diagramma`
- **(d)** L'ho messa in pratica immediatamente, anche in forma grezza → `D4: 4.4 Gesto/prototipo`
- **(e)** L'ho formalizzata: formula, algoritmo, modello → `D4: 4.5 Codice/formalizzazione`

### D5. Quale strumento ti mancherebbe di più se te lo togliessero?

- **(a)** La possibilità di parlare con qualcuno (o di parlare ad alta voce da solo) → `D4: 4.1 Voce`
- **(b)** La possibilità di scrivere (qualsiasi supporto) → `D4: 4.2 Scrittura`
- **(c)** La possibilità di disegnare/schematizzare (lavagna, carta, tablet) → `D4: 4.3 Diagramma`
- **(d)** La possibilità di costruire/toccare/manipolare → `D4: 4.4 Gesto/prototipo`
- **(e)** La possibilità di formalizzare (codice, formule, modelli) → `D4: 4.5 Codice/formalizzazione`

## SEZIONE E — Come ragioni (Forma logica)

### E1. Un collega ti dice: "I nostri migliori clienti sono tutti nel settore manifatturiero." Come reagisci mentalmente?

- **(a)** "Quindi dovremmo concentrarci sul manifatturiero" — la regola è chiara, applico la conclusione → `D5: 5.1 Deduttivo`
- **(b)** "Interessante, ma quanti clienti abbiamo visto? Il campione è abbastanza grande?" — verifico se la generalizzazione regge → `D5: 5.2 Induttivo (verifica)`
- **(c)** "Perché proprio il manifatturiero? Cosa hanno in comune questi clienti che spiega il successo?" — cerco la spiegazione sottostante → `D5: 5.3 Abduttivo`
- **(d)** "Questo mi ricorda quello che succede nel settore X: anche lì i migliori clienti erano tutti in un segmento..." — cerco analogie → `D5: 5.4 Analogico`

### E2. Devi prevedere se una nuova iniziativa funzionerà. Come costruisci la tua previsione?

- **(a)** Parto dai principi: "se vale A e vale B, allora necessariamente..." → `D5: 5.1 Deduttivo`
- **(b)** Guardo i dati storici: "le ultime 5 volte che abbiamo fatto qualcosa di simile, è andata così..." → `D5: 5.2 Induttivo`
- **(c)** Mi chiedo: "qual è lo scenario più plausibile, considerando tutto ciò che so?" → `D5: 5.3 Abduttivo`
- **(d)** Cerco un caso analogo in un altro contesto: "nel settore Y hanno fatto una cosa simile e..." → `D5: 5.4 Analogico`

### E3. Ti trovi di fronte a un risultato inaspettato. La tua prima reazione?

- **(a)** "Qualcuna delle premesse deve essere sbagliata — rivediamole una per una" → `D5: 5.1 Deduttivo`
- **(b)** "Un singolo caso non basta — vediamo se si ripete" → `D5: 5.2 Induttivo`
- **(c)** "Qual è la spiegazione più probabile per questo risultato?" → `D5: 5.3 Abduttivo`
- **(d)** "Dove ho già visto qualcosa del genere? Cosa c'entra questo con...?" → `D5: 5.4 Analogico`

### E4. Come tendi a convincere qualcuno?

- **(a)** Con la logica: "se accetti X, allora devi accettare Y, perché segue necessariamente" → `D5: 5.1 Deduttivo`
- **(b)** Con l'evidenza: "guarda, in tutti questi casi è successo così" → `D5: 5.2 Induttivo`
- **(c)** Con la spiegazione: "il motivo più probabile per cui succede è..." → `D5: 5.3 Abduttivo`
- **(d)** Con l'analogia: "è come quando..." → `D5: 5.4 Analogico`

## SEZIONE F — Gap analysis (Canale ↔ Medium)

### F1. Ripensa a una volta in cui hai capito qualcosa ma non riuscivi a spiegarlo. Cosa è successo?

- **(a)** Lo "vedevo" nella mia mente ma non riuscivo a metterlo in parole → `GAP: D1 Visuo-spaziale → D4 Voce/Scrittura`
- **(b)** Lo "sentivo" come giusto ma non sapevo argomentarlo logicamente → `GAP: D1 Somatico → D4 Voce/Scrittura/Codice`
- **(c)** L'avevo chiaro verbalmente ma non riuscivo a schematizzarlo → `GAP: D1 Verbale → D4 Diagramma`
- **(d)** Non mi capita spesso — di solito riesco a tradurre tra formati → `GAP: nessuno significativo`

### F2. Quando comunichi con un LLM (ChatGPT, Claude, ecc.), ti capita di sentirti frustrato perché...

- **(a)** Vorrei disegnare uno schema ma posso solo scrivere testo → `GAP: D4 preferito Diagramma → D4 disponibile Scrittura`
- **(b)** Vorrei discuterne a voce — scrivere prompt mi rallenta → `GAP: D4 preferito Voce → D4 disponibile Scrittura`
- **(c)** Le mie intuizioni sono "sensazioni" che non so trasformare in prompt → `GAP: D1 Somatico → D4 Scrittura`
- **(d)** Non ho particolari frustrazioni — mi trovo bene a comunicare per iscritto → `GAP: nessuno significativo`

### F3. In quale situazione senti che il tuo pensiero perde qualità?

- **(a)** Quando devo scrivere invece di parlare → `GAP: Voce → Scrittura`
- **(b)** Quando devo parlare invece di scrivere → `GAP: Scrittura → Voce`
- **(c)** Quando devo spiegare a parole quello che nella mia testa è un'immagine o schema → `GAP: Visuo-spaziale → Verbale`
- **(d)** Quando devo formalizzare quello che "sento" essere giusto → `GAP: Somatico/Intuitivo → Formale`
- **(e)** Quando non posso costruire/prototipare e devo solo descrivere → `GAP: Gesto → Verbale/Scritto`

## SCORING

### Come calcolare il profilo

**Per ogni dimensione**, conta quante volte hai scelto ciascun valore:

#### D1 — Canale cognitivo (Sezione A, 5 domande)
| Valore | Risposte (a) | Risposte (b) | Risposte (c) | Risposte (d) | Risposte (e) |
|--------|:---:|:---:|:---:|:---:|:---:|
| 1.1 Verbale | _ | | | | |
| 1.2 Visuo-spaziale | | _ | | | |
| 1.3 Somatico | | | _ | | |
| 1.4 Sonoro | | | | _ | |
| 1.5 Astratto | | | | | _ |

**Dominante:** il valore con più risposte. **Secondario:** il secondo più alto.

#### D2 — Strategia (Sezione B, 7 domande)
Conta le ricorrenze. I valori con 2+ occorrenze sono le tue strategie dominanti.

#### D3 — Direzione (Sezione C, 5 domande)
Conta le ricorrenze. Nota se tendi a divergere, convergere, o ad alternare (ciclo).

#### D4 — Medium (Sezione D, 5 domande)
Conta le ricorrenze. Nota separatamente il destinatario principale (D3).

#### D5 — Forma logica (Sezione E, 4 domande)
Conta le ricorrenze.

#### Gap Analysis (Sezione F)
Le risposte indicano dove perdi informazione nella traduzione pensiero→comunicazione. Confronta D1 dominante con D4 dominante: se sono diversi, hai un gap da gestire.

### Come leggere il profilo

Il profilo si esprime come stringa: `D1.valore | D2.valore | D3.valore | D4.valore | D5.valore`

Esempio: `Visuo-spaziale | Adversariale+Socratico | Laterale+Convergente | Diagramma+Voce | Abduttivo`

Per l'interpretazione dettagliata, vedi: `interpretazione_profili.md`
